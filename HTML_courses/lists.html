HTML Lists

Introduction

Lists let you group related items in HTML. They make your page easier to read and help structure information like steps, features, or options.

Unordered Lists <ul>

Use <ul> for items where order doesn't matter. Each item is inside <li> tags.

<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Eggs</li>
</ul>
  • Milk
  • Bread
  • Eggs

Ordered Lists <ol>

Use <ol> for items that must be in order (like steps). You can change number style with the type attribute.

<ol type="1">
  <li>Step One</li>
  <li>Step Two</li>
</ol>

<ol type="A">
  <li>Alpha</li>
  <li>Beta</li>
</ol>

<ol type="i">
  <li>First</li>
  <li>Second</li>
</ol>
  1. Step One
  2. Step Two
  1. Alpha
  2. Beta
  1. First
  2. Second

Description Lists <dl>

Use <dl> for pairs of <dt> (term) and <dd> (description) – great for glossaries.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
HTML
HyperText Markup Language
CSS
Cascading Style Sheets

Nested Lists

Put lists inside other lists to show subcategories.

<ul>
  <li>Groceries
    <ul>
      <li>Milk</li>
      <li>Bread</li>
    </ul>
  </li>
  <li>To-Do
    <ol>
      <li>Clean room</li>
      <li>Do homework</li>
    </ol>
  </li>
</ul>
  • Groceries
    • Milk
    • Bread
  • To-Do
    1. Clean room
    2. Do homework

Customizing Lists with CSS

Change bullets or numbers with list-style-type.

<ul style="list-style-type: disc;">...</ul>
<ul style="list-style-type: circle;">...</ul>
<ul style="list-style-type: square;">...</ul>
<ul style="list-style-type: none;">...</ul>
<ol style="list-style-type: upper-roman;">...</ol>
  • Disc
  • Style
  • Circle
  • Style
  • Square
  • Style
  • No bullets
  • Still a list

  2. Second

Best Practices

  • Use <ul> for unordered items.
  • Use <ol> for ordered steps.
  • Use <dl> for terms and definitions.
  • Always wrap list items with <li> (except in <dl>).
  • Don’t use lists for layout—use them for actual lists.

Common Mistakes

<ul> Apples Oranges </ul>
← Missing <li> tags!
<ul>
<li>Apples</li>
<li>Oranges</li>
</ul>
<ol> <li>Step One</li> <li>Step Two <li>Substep</li> </li> </ol>
← Nested <li> without a new <ol>!
<ol>
<li>Step One</li>
<li>Step Two <ol> <li>Substep</li> </ol> </li>
</ol>

Challenge Project: Recipe Page

  • Main heading for the recipe title.
  • Ingredients as an unordered list.
  • Steps as an ordered list.
  • Glossary of terms as a description list.
<h1>Chocolate Cake</h1>

<h2>Ingredients</h2>
<ul>
  <li>Flour</li>
  <li>Sugar</li>
  <li>Cocoa powder</li>
</ul>

<h2>Steps</h2>
<ol>
  <li>Preheat oven</li>
  <li>Mix ingredients</li>
  <li>Bake for 30 minutes</li>
</ol>

<h2>Glossary</h2>
<dl>
  <dt>Preheat</dt>
  <dd>Heat the oven before baking</dd>
  <dt>Mix</dt>
  <dd>Combine ingredients together</dd>
</dl>

Quick Recap: Cheat Sheet

  • <ul>: Unordered (bulleted) list
  • <ol>: Ordered (numbered/lettered) list
  • <dl>: Description (terms + definitions) list
  • <li>: List item
  • <dt>: Term in description list
  • <dd>: Definition in description list
