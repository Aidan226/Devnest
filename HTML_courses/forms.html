HTML Forms

Basic Form Structure

Forms are created with the <form> element. Use action for the destination and method for how data is sent. 

<form action="/submit" method="post">
  ...
</form>

Text Inputs & Textarea

  • <input type="text"> for single-line
  • <textarea> for multi-line
<input type="text" name="username" placeholder="Enter your name" maxlength="20" value="Alice">
<textarea name="bio" rows="3" placeholder="Tell us about yourself"></textarea>

Password Input

<input type="password" name="pass">

Email, Number, URL, Tel, Date/Time

<input type="email" placeholder="Email">
<input type="number" min="0" max="100">
<input type="url" placeholder="https://...">
<input type="tel" placeholder="123-456-7890">
<input type="date">

Radio Buttons

Choose one option only.

<input type="radio" name="color" value="red"> Red
<input type="radio" name="color" value="blue"> Blue

Checkboxes

Pick as many as you want.

<input type="checkbox" name="like" value="pizza"> Pizza
<input type="checkbox" name="like" value="icecream"> Ice Cream

Drop-down Menus

<select name="country">
  <option>USA</option>
  <option>UK</option>
</select>

Buttons

  • <input type="submit"> (submit form)
  • <input type="reset"> (clear form)
  • <button> (custom button)
<input type="submit" value="Send">
<input type="reset" value="Clear">
<button>Regular Button</button>

Labels for Accessibility

  • Use <label for="id"> for every input (the for attribute matches the input’s id).
  • Labels help screen readers and improve usability.
<label for="email">Email:</label>
<input id="email" type="email">

File Upload

<input type="file" name="myphoto">

Hidden Fields

<input type="hidden" name="userid" value="123">
(Hidden fields are not visible to users)

Fieldset & Legend

<fieldset>
  <legend>Contact Info</legend>
  ...
</fieldset>
Contact Info

Validation: Required, Patterns, Min/Max

  • required — must be filled in
  • pattern — regex pattern
  • min and max for numbers
<input type="email" required>
<input type="text" pattern="[A-Za-z]+">
<input type="number" min="1" max="10">

Accessibility Tips

  • Always use <label> for every input.
  • Placeholders are not a replacement for labels.
  • Make sure inputs are large enough for easy use.

Styling Forms with CSS

  • Use padding, border-radius, and box-shadow for inputs.
  • Highlight inputs on :focus.
  • Use display: flex or grid for layout.
  • Style buttons with color and hover effects.
input, select, textarea {
  padding: 0.7em 1em;
  border-radius: 8px;
  border: 1.5px solid #1c2a4e;
}
input:focus {
  border-color: #5ee7ff;
  box-shadow: 0 0 0 3px #5ee7ff44;
}
button:hover {
  background: #7c4dff;
  color: #ffe066;
}

Common Mistakes

<input type="text"> ← No label!
<input type="text" style="width:50px;"> ← Too small!
<input type="email"> ← No name attribute!
<label for="user">Username:</label> <input id="user" name="user" type="text">

Interactive Activity

  1. Create a login form with username & password fields and a submit button.
  2. Make a survey with radio buttons and checkboxes.
  3. Make a registration form with email, password, and country select.
<!-- Login Form -->
<form>
  <label for="loginuser">Username:</label>
  <input id="loginuser" name="loginuser">
  <label for="loginpass">Password:</label>
  <input id="loginpass" type="password" name="loginpass">
  <input type="submit" value="Login">
</form>

<!-- Survey -->
<form>
  Favorite Pet:
  <input type="radio" name="pet" value="cat"> Cat
  <input type="radio" name="pet" value="dog"> Dog
  
Hobbies:
  <input type="checkbox" name="hobby" value="reading"> Reading
  <input type="checkbox" name="hobby" value="sports"> Sports
</form>

<!-- Registration -->
<form>
  <label>Email: <input type="email" name="regemail"></label>
  <label>Password: <input type="password" name="regpass"></label>

</form>
